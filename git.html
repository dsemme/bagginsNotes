GIT NOTES

1) Link de download do git: Git Download. Checar versão [git --version]

2) Comandos de configuração do Git:

  • git config --global user.name "username"
  • git config --global user.email "user@email.com"
  • git config --global core.editor "editor" !important1
    •

!important1 - Para configurar o vscode como editor padrão utilizado pelo Git, serão necessárias algumas configurações adicionais:

  • git config --global core.editor "code --wait"
  • git config --global -e
    •

    Após abrir o arquivo de configuração do git, adicionar as seguintes linhas:

  • [diff]
    • tool = default-difftool
  • [difftool "default-difftool"]
    • cmd = code --wait --diff $LOCAL $REMOTE

3) Iniciando o projeto no git

  • git init (cria um repositório em .git)

Status dos arquivos no diretório e no repositório:

  • untracked: arquivos fora do repositório
  • unmodified: adicionado ao repositório, mas não modificado
  • modified: modificados dentro do repositório
  • staged: modificados, readicionados e prontos para serem commitados

Adicionando ao repositório (staged)

  • git add file.ext
    •

Commitando projeto

  • git commit -m "Descrição do tipo de modificação realizada no projeto"
  • git commit -am (commita todos arquivos modificados e adiciona msg)

Visualizando o Status do repositório

  • git status
    •

3) Visualizando logs

Mostra todas as modificações realizadas no repositório.

  • git log
  • git shortlog
  • git shortlog -sn (mostra somente nº de commits e author)
  • git log --graph (gráfico de branchs)
  • git show "hash" (mostra os detalhes que foram recentemente add)
  • git diff (mostra atualizações antes de commitar)
  • git diff --name-only (mostra somente arquivos modificados)

4) Fazendo alterações e desfazendo ações no repositório

  • git checkout file.ext (desfaz alterações nos arquivos unstaged)
  • git reset HEAD file.ext (retira arquivos staged e permite aplicar checkout após o processo)
  • git reset --soft "penultimaHashCommitada" (retorna ao penúltimo commit e coloca o último de volta em staged)
  • git reset --mixed "penultimaHashCommitada" (retorna ao penúltimo commit e coloca o último em modified)
  • git reset --hard "penultimaHashCommitada" (desfaz todas as alterações do último commit)

5) Conectando Git ao GitHub utilizando SSH

Gerando a chave pública de acesso

  • ssh-keygen -t rsa -b 4096 -C "user@email.com" (no prompt de comando)
  • Coloca a chave gerada no link SSH and GPG keys do github.
    •

Conectando repositório local ao remoto

  • git remote add origin "userongithub" (repositórios com o mesmo nome)
  • git push -u origin master (envia tudo do local pro remoto. necessário commit de todo repositório)
  • git remote -v (informações do repositório remoto)
  • git push origin master (atualiza repositorio remoto no branch especificado)

Clonando o repositório

  • git clone git@github.com:dsemme/bagginsNotes.git novo-nome

6) Trabalhando com Branchs

  • git checkout -b nomedobranch (cria e ja modifica o ponteiro para o novo branch)
  • git branch (lista os branchs e mostra qual está em uso)
  • git checkout nomedobranch (alterna o uso para outro branch)
  • git branch -D nomedobranch (deleta um branch)

7) .gitignore

  • Cria um arquivo .gitignore no repositório
  • Adiciona os arquivos que não serão trackeados ou a extensão dos arquivos (ex: *.xls , nomes.json)

8) Outros comandos importantes

STASH
  • git stash (deixa arquivos modified em standby - WIP (work in progress))
  • git stash apply (aplica as mudanças que estavam em standby)
  • git stash list (lista tudo que tá em stash)
  • git stash clear (limpa o stash)
ALIAS
  • git config --global alias.nomeDoAlias comandoAlvo (ex: git config --global alias.s status)
TAGS

    Procedimento realizado após cada novo push para o github

  • git tag -a versaoDaTag -m "descDaTag" (git tag -a 1.0.0 -m "Primeiro release.")
  • git push origin master --tags (push das tags)
  • git tag (lista todas as tags)
  • git tag -d nomeDaTag (deleta a tag selecionada do repositorio local)
REVERT
  • git revert hashDoCommit (desfaz as ultimas alterações do arquivo mantendo o histórico)
DELETANDO BRANCHS, TAGS... DO REPOSITÓRIO REMOTO
  • git push origin :nomeDoElemento (ex: git push origin :1.0.0 , git push origin :branch)
